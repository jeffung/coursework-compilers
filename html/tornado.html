Tornado

A similar web application to node.js is Tornado, which is a non-blocking web application framework written in python. Facebook acquired a company called FriendFeed, which was the company that Tornado was actually developed for.

It is implemented in a very similar way to node.js. It uses one primary thread of execution, and all other processes are asynchronously non-blocking, by way of a callback function. This makes Tornado and node.js very similar in terms of use styles and required experience.

When used to solve the problem of handling 10000 concurrent connections, Tornado performs quite well compared to other Python based servers, handling between 3500 per second on a single thread, and up to 8000 using for nginx frontends. Node.js is, however, still better.

The main benefit of using Tornado is if your specialty is in python, or the majority of your intended application is in python. There are fewer modules available for Tornado, than there are for node.js however. This is likely due to the massive popularity of node.js and JavaScript in general, in web application development circles.
